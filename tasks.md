# Рекомендации по работе с задачами

1. Новая задача сначала добавляется в BACKLOG.
2. При начале работы задача переносится в IN PROGRESS.
3. После завершения разработки — в REVIEW / TESTING.
4. После успешного тестирования — в DONE.
6. Если в процессе тестирования выявлены проблемы:
   - фиксируются в разделе "Комментарии"
   - при необходимости создаётся новая задача типа Bug.
8. Мой совет, делайте фиксации(коммиты по чаще плиззз)



# Типы задач
- Новый функционал - Что-то новое в системе.
- Баг - Что-то сломалось или работает не так, как должно.
- Доработка - Улучшение уже существующего функционала.
- Инфраструктура - Docker, nginx, БД, переменные окружения, деплой и т.д.
- Документация - описание проекта, обновление README, схемы, инструкции.
