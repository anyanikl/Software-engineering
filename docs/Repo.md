1) backend
    - в общем это бэк, работает самостоятельно и живет в своем контейнере. Внутри есть два решения Api и Dto. 
        - FunApi - собственно говоря сам бэк. С базовой структурой контроллер, сервисы, бд, ну и настроенные мной уже конфиги. Содержит:
            - контроллеры (API endpoints)
            - сервисы (бизнес-логика)
            - модели и контекст базы данных
            - конфигурации (auth, cors, environment, middleware)
        - FunDto - бэк но не совсем. Это так называемые Dto. Определенный контракт по которому будут общаться front и back. Содержит:
            - Dto в папке Models - к базе не конктиться, про неё ничего не знает, просто доп прослайка, определяющая контракт общения(передачи)
